# 家計・資金管理アプリ（計画 vs 実績）

マネーフォワード ME のデータを取り込み、**月の計画（利用上限）と実績を色分けで見える化**する Web アプリです。
「ここまでは使ってよい」を日割りで把握し、ムダを省くことを目的にしています。

## コンセプト（3階層の資金管理）

```
📅 年次   → 年初残高 ＋ 毎月の収支を積み上げ、現金残高の推移を監視
🗓 日次   → 月内の日別スプレッドシート（入出金と残高の推移）
🎯 月次   → 計画（上限）をエクセル/CSVで一括アップロード、実績はMF CSVで自動取込
```

## 主な機能

| タブ | 内容 |
|------|------|
| 📊 ダッシュボード | 残り日数 / 今日まで使ってよい額（日割り）/ 計画 vs 実績 vs 着地予測 / カテゴリ別の色分け |
| 📅 年次（残高推移） | 年初残高の設定、月別の入金・支出・収支・累計残高、年末残高予測 |
| 🗓 日次一覧 | 日付×カテゴリのスプレッドシート、日次収支・残高推移（緑＝入金/赤＝支出） |
| 🎯 月の計画 | **エクセル/CSVで計画を一括アップロード**（テンプレートDL対応、複数月まとめて）＋手入力 |
| 📥 実績取込 | MF ME の CSV を取込（Shift-JIS/UTF-8 自動判定・重複登録防止） |
| 🧾 明細 | 当月の明細一覧、現金の手入力、削除 |
| ⚙️ 項目設定 | **入金・出金の項目を画面から自由に追加・編集・並べ替え**。ここで作った項目が日次一覧の列になる |

### 柔軟な項目マスタ（中小企業向けの肝）

会社・用途ごとにカード・ローン・科目はバラバラ。そこで固定カテゴリではなく、
**ユーザーが「項目」を自由に定義** できる設計にしています。

```
項目マスタ（⚙️項目設定タブで管理）
　├─ 区分：入金 / 出金
　├─ グループ：現金 / カード / ローン / その他…（任意）
　├─ 名前：「JCB」「SMBC」「給与」など自由
　└─ 表示順・色
        ↓
日次一覧の「列」を動的に生成
```

これにより、手元のエクセルの列構成（給与・現金・ローン・JCB・UC・SMBC…）を画面上で再現でき、
将来は会社ごとに項目を作り替えるだけで使い回せます。

### 計画の一括アップロード

「🎯 月の計画」タブで **テンプレートCSVをダウンロード → エクセルで編集（コピペでOK）→ アップロード**。
1ファイルで複数月を一括登録できます。列構成：

```
年月, 現金上限, カード上限, 入金目標, 食費, 日用品, 交通費, ...
2026-01, 30000, 200000, 400000, 40000, ...
2026-02, ...
```

## 使い方

1. `index.html` をブラウザで開く（スマホでも可）
2. 初回はメールアドレスで **新規登録**（確認メールが届く場合はリンクを開く）→ ログイン
3. 「🎯 月の計画」で現金・カードの上限を入力して保存
4. マネーフォワード ME →「家計簿」→「入出力」→ CSV ダウンロード
5. 「📥 実績取込」にその CSV をドラッグ＆ドロップ
6. 「📊 ダッシュボード」で計画との差を確認

## データ構成（Supabase / PostgreSQL）

すべて `expense_` 接頭辞で、既存データとは完全分離。

| テーブル | 役割 |
|---------|------|
| `expense_categories` | カテゴリマスタ |
| `expense_year_settings` | 年次設定（年初の現金残高） |
| `expense_monthly_plans` | 月次計画（現金/カード上限・入金目標・月初残高） |
| `expense_category_budgets` | カテゴリ別の上限 |
| `expense_transactions` | 実績明細（MF/CSV/手入力） |
| `expense_import_log` | 取込履歴 |

- RLS 有効。ログイン済みユーザーのみアクセス可（匿名アクセス不可）。
- 公開キー（`sb_publishable_...`）はブラウザ用の公開鍵で、コードへの埋め込みは想定どおり。
